将 10 个顶级思维模式 转换成适用于 大语言模型（LLM） 的 Prompt，并在 Prompt 中设计一些灵活的变量（例如：文章、链接、主题、问题等），让你能够根据需要插入具体的内容，从而灵活应用这些思维模式。

使用说明
- 变量部分：
    - {内容描述或链接}：可以是具体文章、项目、流程、问题或主题。
    - {分析目标}：明确分析的具体目标，比如“寻找问题根源”、“制定优化流程”等。
- 输出逻辑：
    - 每个 Prompt 的输出都经过结构化分步骤处理，方便整理和应用。

---
1. SWOT 分析法
```
请基于以下内容进行 SWOT 分析：

内容：{内容描述或链接}
目标：{分析目标，如个人成长、企业战略、项目优化等}

要求：
1. 识别该内容中的 **优势 (Strengths)**：它有哪些值得肯定或有利的因素？
2. 指出 **劣势 (Weaknesses)**：它有哪些需要改进或薄弱的地方？
3. 探索 **机会 (Opportunities)**：有哪些潜在的发展机遇或外部有利条件？
4. 评估 **威胁 (Threats)**：有哪些可能的风险或外部挑战？

请按照“优势、劣势、机会、威胁”顺序清晰地输出每个部分的分析结果。
```

2. 六顶帽思维
```
请基于以下内容运用 “六顶帽思维” 进行全面分析：

内容：{内容描述或链接}
目标：{分析目标，如探讨某问题、制定决策、评估策略等}

要求：
1. **白帽思维**：列出事实与数据（明确可核实的信息）。
2. **红帽思维**：描述对该内容的直觉或情感反应。
3. **黑帽思维**：提出该内容中的潜在风险或可能出现的问题。
4. **黄帽思维**：思考该内容的积极面，找出其潜在优势或好处。
5. **绿帽思维**：发散思维，提出创新的想法或解决方案。
6. **蓝帽思维**：从整体上总结该分析的结果，并提供明确的下一步行动建议。

请以六顶帽的顺序清晰地输出分析内容。
```

3. 本质追溯法
```
请基于以下内容运用“本质追溯法”进行深度分析：

内容：{内容描述或链接}
目标：{分析目标，如探究问题本质、理解关键原因、制定解决方案等}

要求：
1. 运用 “为什么” 的方式，依次追问以下问题：
   - 为什么会发生？
   - 为什么没有发现？
   - 为什么没有预测？

2. 在每个问题后，进一步追问其背后的原因，直到找到最根本的本质。

请分步骤逐层输出追问的结果，最终总结该内容的核心问题本质和可能的解决方案。
```

4. 五遍递归法
```
请运用“五遍递归法”对以下内容进行全面分析：

内容：{内容描述或链接}
目标：{分析目标，如梳理关键逻辑、理解核心问题、优化解决方案等}

要求：
1. 第 1 遍：简要描述内容的主要信息或事件。
2. 第 2 遍：挖掘内容的主要目标或意图。
3. 第 3 遍：梳理内容中的关键细节，找出隐藏的潜在信息。
4. 第 4 遍：深入分析内容的背景、上下文及其逻辑联系。
5. 第 5 遍：总结内容的本质规律，并提出实际的应用建议或解决方案。

请按照五遍递归的顺序逐步输出，确保每一遍的结果有逻辑递进关系。
```

5. PDCA 循环
```
请基于以下内容运用 PDCA 循环方法进行分析：

内容：{内容描述或链接}
目标：{分析目标，如改进流程、优化项目管理、提升执行效果等}

要求：
1. **Plan - 计划阶段**：
   - 识别内容中的主要目标和问题。
   - 制定明确的计划（包括目标、策略和指标）。

2. **Do - 执行阶段**：
   - 描述计划中的关键执行步骤。
   - 指出需要落实的具体行动。

3. **Check - 检查阶段**：
   - 评估执行过程中的成果和问题。
   - 分析与预期目标的差距。

4. **Act - 处理阶段**：
   - 提出改进建议或调整措施。
   - 形成下一步的行动计划。

请分阶段清晰输出每个部分的结果，并保证逻辑连贯。
```

6. 效率分析法
```
请基于以下内容运用“效率分析法”对任务进行梳理和优化：

内容：{内容描述或链接}
目标：{分析目标，如提升任务效率、优化资源配置、改进流程等}

要求：
1. **收集**：列出任务中的所有要素或步骤。
2. **整理**：对任务进行分类，找出需要优先处理的部分。
3. **组织**：设计合理的下一步行动计划，确保任务逻辑清晰。
4. **执行**：描述执行的具体步骤和需要关注的关键点。
5. **回顾**：分析任务执行过程中的不足，并提出改进建议。

请逐步输出每个阶段的内容，确保逻辑清晰、步骤完整。
```

7. 心流思维
```
请运用“心流思维”设计一套明确的任务流程，让内容能够更高效地完成：

内容：{内容描述或链接}
目标：{分析目标，如找到进入心流状态的方法、提高专注力、完成复杂任务等}

要求：
1. **明确目标**：将内容拆解为清晰的小目标，确保难度适中。
2. **构建反馈**：设计任务中的即时反馈机制，帮助实时评估进展。
3. **优化专注**：制定能够屏蔽干扰的方法，并专注于核心任务。
4. **高效时间块**：规划 25 分钟的高专注时间，用于实现心流。
5. **回顾奖励**：回顾任务完成情况，并为完成部分设立适当的奖励机制。

请分步骤输出如何引导进入心流状态，并根据目标调整分析结果。
```

8. 七问分析法
```
请基于“七问分析法”对以下内容进行全面分析：

内容：{内容描述或链接}
目标：{分析目标，如理解问题核心、设计解决方案、制定策略等}

要求：
1. **What - 什么**：明确内容的核心目标或主要问题是什么？
2. **Why - 为什么**：分析为什么需要完成这个目标或解决这个问题？
3. **Who - 谁**：指出与目标或问题相关的关键人物或团队。
4. **When - 何时**：明确关键的时间节点或时间管理要求。
5. **Where - 何地**：分析事件或问题发生的环境或场所。
6. **How - 如何**：提出解决问题或实现目标的具体方法。
7. **How much - 多少**：评估所需的资源投入或目标达到的程度。

请逐步回答上述七个问题，确保逻辑清晰、内容全面。
```

9. 反思思维
```
请基于“反思思维”对以下内容进行回顾和总结：

内容：{内容描述或链接}
目标：{分析目标，如总结经验教训、优化后续行动、提升认知能力等}

要求：
1. **记录**：回顾今天发生的内容或事件，并记录关键点。
2. **启发**：分析这件事对其他领域或未来的启示。
3. **行动**：根据反思提出改进建议，明确下一步行动。
4. **总结**：从整体上总结核心经验，并探讨可复制的方法。

请分步骤输出每个部分的反思结果，确保总结具体且具有行动价值。
```
